## 分析

### CLOCK_WATCHDOG_TIMEOUT とは

CPU コアが指定時間内にクロック割り込みに応答しなかった場合に発生する。
パラメータ1 はハングした論理プロセッサ番号を示す。

### 論理プロセッサ #12 の位置

i7-11700K は 8コア/16スレッド（LP#0〜15）:
- LP#12 = **コア6のハイパースレッド**
- 同じコア（コア6）が繰り返し問題を起こしている

### 推定原因（優先度順）

1. **CPU コア6の物理的劣化**
   - Rocket Lake (14nm) は消費電力・発熱が高く、経年劣化しやすいアーキテクチャ
   - 特定コアのみ繰り返しハングするのは物理的損傷の典型パターン

2. **電圧不足 (Vcore)**
   - ブースト時にコア6への供給電圧が不足している可能性
   - BIOS で Vcore を +0.025〜0.05V オフセットすることで改善する場合がある

3. **オーバークロック（該当する場合）**
   - OC設定があれば即座に定格へ戻す

4. **冷却不良**
   - コア間の温度差が大きい場合、特定コアの冷却が不十分な可能性

## 推奨対策

### 即時（応急処置）

- [ ] **BIOS でオーバークロックを無効化**（設定がある場合）
- [ ] **BIOS を最新バージョンに更新**（Intel マイクロコード更新が含まれることがある）
- [ ] **HWiNFO でコア別温度を監視** — コア6が他より著しく高温でないか確認

### 短期（安定運用）

- [ ] **BIOS でコア6を無効化** — 14スレッド（7C/14T）で動作するが BSoD は回避できる
  - 設定場所: Advanced > CPU Configuration > Active Processor Cores（マザーボードにより異なる）
  - 一部マザーボードでは個別コア無効化に対応していない場合がある

### 長期

- [ ] **CPU の交換を検討** — i7-11700K が経年劣化している場合、根本的な解決は交換のみ

---

## PokeScouter プロジェクトへの影響

- **OCR セットアップ自体は BSoD の原因ではない**（venv 作成は軽量処理）
- ただし PyTorch / PaddlePaddle で **全コアに高負荷がかかる処理**を行うと再発リスクあり
- GPU 処理（CUDA）は CPU コア問題とは独立しているが、CPU 側の前処理でコア6が使われると BSoD になる可能性がある
- **対策完了まで**: 長時間の CPU 集約処理（モデルのコンパイル等）は注意して実施する
