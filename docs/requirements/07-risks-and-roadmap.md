# リスク・ロードマップ

## 8. リスク・課題

| リスク                        | 影響度 | 対策                                                      |
| ----------------------------- | ------ | --------------------------------------------------------- |
| ChampionsのUIが未確認         | 高     | M0で最優先確認。テンプレート・UI座標はconfig化            |
| OCR精度が実用レベルに達しない | 高     | 3エンジン比較 + 辞書照合補正 + Azure Visionで精度上限測定 |
| ダメージ計算式がSVと異なる    | 中     | 有志のChampions向け検証結果を参照                         |
| WebSocketフレーム送信が重い   | 中     | M1で実測後に最適化方針決定                                |
| 多言語ポケモン名              | 中     | 日→韓→英→中のフォールバックOCR                            |
| ゲームアップデートでUI変更    | 中     | テンプレート・座標のconfig化で更新を容易に                |
| 利用規約との関係              | 低     | 利用規約・大会レギュレーションを確認                      |

## 9. 開発ロードマップ（Phase 1）

| マイルストーン | 内容                                                              | 目安  |
| -------------- | ----------------------------------------------------------------- | ----- |
| M0             | 実機UI確認、OCRエンジン精度テスト、テンプレート画像収集、環境構築 | 1週間 |
| M1             | ブラウザ映像表示 + FastAPIサーバー + WebSocket疎通                | 1週間 |
| M2             | フレーム送信 → 場面識別 → 結果返却                                | 1週間 |
| M3             | 選出画面6体認識（GLM-OCR）+ 対戦画面ポケモン名/HP OCR             | 2週間 |
| M4             | ダメージ計算エンジン実装・検算 + REST API化                       | 2週間 |
| M5             | 映像 + 認識結果 + ダメージ計算の統合UI                            | 2週間 |
| M6             | OCR精度改善、WebSocket最適化、実戦テスト                          | 2週間 |

合計見積もり：約10〜12週間
