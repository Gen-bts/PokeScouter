# プラットフォーム・API設計

## Web UI + Pythonバックエンドのハイブリッド構成

```
ブラウザ (Frontend)                Python Server (Backend)
┌───────────────────┐             ┌──────────────────────┐
│ 映像キャプチャ     │  WebSocket  │ OCRエンジン          │
│ (mediaDevices)    │────────────→│ ├ PaddleOCR   ┐      │
│                   │             │ ├ manga-ocr   ├ CUDA │
│ 映像表示 (Canvas) │             │ └ GLM-OCR     ┘      │
│                   │←────────────│                      │
│ ダメージ結果表示   │  WebSocket  │ ダメージ計算エンジン  │
│                   │             │                      │
│ パーティ登録UI     │  REST API   │ 3層データベース       │
│ 設定画面          │────────────→│ (base/override/season)│
└───────────────────┘             └──────────────────────┘
                                     RTX 5070 (12GB VRAM)
```

## 映像フロー

1. ブラウザがキャプチャーボード映像を取得・表示
2. 解析用フレームをWebSocketでPythonに送信（1〜5fps）
3. Python側で場面識別 → OCR → ダメージ計算
4. 結果をWebSocketでブラウザに返却・表示

※フレーム送信の最適化方針（クロップ・圧縮・頻度）はM1で実測後に決定。

## API設計

| エンドポイント                  | 用途                                     |
| ------------------------------- | ---------------------------------------- |
| `ws://localhost:8000/ws/battle` | 映像フレーム送受信・リアルタイム認識結果 |
| `/api/pokemon`                  | ポケモンデータ取得                       |
| `/api/my-pokemon`               | ユーザー登録ポケモンCRUD                 |
| `/api/damage-calc`              | 手動ダメージ計算                         |
| `/api/season`                   | シーズン情報                             |
