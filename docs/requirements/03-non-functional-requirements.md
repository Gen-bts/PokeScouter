# 非機能要件

## 4.1 パフォーマンス

| 項目                        | 目標値                    |
| --------------------------- | ------------------------- |
| 場面識別                    | 500ms以内                 |
| ポケモン認識（選出画面6体） | 3秒以内                   |
| ポケモン名OCR（対戦中）     | 100ms以内                 |
| ダメージ計算（1組）         | 50ms以内                  |
| メモリ使用量                | 512MB以下（GPU VRAM別途） |

## 4.2 ユーザビリティ

- キャプチャ映像の近くに結果を表示（視線移動を最小化）
- 映像上にオーバーレイを重ねない
- 詳細情報はホバー/タップで表示（情報過多の防止）
- ダメージ深刻度を色の濃淡で表現

## 4.3 拡張性

- Phase 2〜4 の機能追加を見据えたモジュラー設計
- OCRエンジンを抽象化し差し替え可能に
- データ・テンプレート画像の追加が容易な構造

## 4.4 信頼性

- OCR誤認識時の手動修正UI
- 認識信頼度が低い場合は候補リストを提示
- 定期的な自動保存
