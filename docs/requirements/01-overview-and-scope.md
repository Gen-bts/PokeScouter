# 概要・スコープ

## 1. プロジェクト概要

キャプチャーボード経由で取得した Pokémon Champions の対戦映像を画像認識で解析し、ダメージ計算を自動化するツール。Web UI + Pythonバックエンドのハイブリッド構成。

### 1.1 先行事例からの技術的知見

| 項目                 | 手法                                           |
| -------------------- | ---------------------------------------------- |
| ポケモン名の認識     | OCR + 辞書照合（多言語フォールバック）         |
| ポケモン立ち絵の識別 | OpenCV テンプレートマッチング                  |
| 場面識別             | 特定UI要素の有無をテンプレートマッチングで判定 |
| HP読み取り           | 自分側：OCR、相手側：HPバーのピクセルカウント  |
| ダメージ計算の高速化 | 乱数の重複をまとめた連想配列方式               |

### 1.2 Champions 特有の考慮事項

- 新規タイトルのためUI・フォント・配色が未確認（M0で最優先確認）
- メガシンカの復活（計算式への反映が必要）
- ダメージ計算式がSVと異なる可能性（有志の検証結果を参照）
- ゲーム配信直後のためポケモンデータの整備が必要

## 2. スコープ定義

| フェーズ          | 内容                                             | 優先度 |
| ----------------- | ------------------------------------------------ | ------ |
| Phase 1（本要件） | 画像認識による自動ダメージ計算（シングルバトル） | 最優先 |
| Phase 1.5         | ダブルバトル対応                                 | 高     |
| Phase 2           | 対戦記録・ログ保存                               | 高     |
| Phase 3           | 選出アドバイス・相性評価                         | 中     |
| Phase 4           | 統計分析・戦績ダッシュボード                     | 低     |

### Phase 1 の対象外

ダブルバトル、スマホカメラ入力、スクリーンショット手動アップロード、育成支援、対戦記録
