# 機能要件

## 3.1 映像入力（FR-001）

| 項目           | 内容                                               |
| -------------- | -------------------------------------------------- |
| 入力方式       | USB キャプチャーボード（HDMI → USB）               |
| 映像取得       | ブラウザの `navigator.mediaDevices.getUserMedia()` |
| 対応解像度     | 1920×1080、1280×720                                |
| フレームレート | 映像表示: 30fps、解析用: 1〜5fps（可変）           |

## 3.2 場面識別（FR-002）

| 場面     | 判定方法                           | アクション                     |
| -------- | ---------------------------------- | ------------------------------ |
| 選出画面 | 固有UI要素のテンプレートマッチング | 相手パーティ認識を開始         |
| 対戦画面 | Lv表示・HPバー等の検出             | ポケモン認識・ダメージ計算開始 |
| その他   | 上記に該当しない                   | 待機                           |

## 3.3 ポケモン認識（FR-003）

**選出画面：** ポケモンアイコン/立ち絵のテンプレートマッチングで相手6体を識別。2回連続一致で確定。

**対戦画面：** ポケモン名のOCR + 辞書照合。日本語→ハングル→英語の順にフォールバック。交代検知時に実行。

## 3.4 HP読み取り（FR-004）

| 対象 | 手法                         | 精度目標   |
| ---- | ---------------------------- | ---------- |
| 自分 | HPバー内数値のOCR            | 正確な数値 |
| 相手 | HPバーの有色ピクセルカウント | ±2%以内    |

## 3.5 ダメージ計算（FR-005）

**入力：** 攻撃側/防御側ポケモン情報（自分側：事前登録、相手側：使用率上位の型を仮定）、場の状態・ランク補正・メガシンカ状態（初期は手動切替）

**出力：** ダメージ範囲、確定数、致死率（乱数16通り全パターン考慮）、使用率上位の技別表示

**計算ロジック：**
- Champions向けの有志検証結果 + ポケモンWiki等を参照
- 連続技は重複ダメージをまとめた連想配列方式で高速化
- メガシンカ時の種族値・特性・タイプ変化に対応

## 3.6 パーティ登録（FR-006）

手動入力 or ボックス画面のOCR読み込み。ポケモン名・性格・努力値・個体値・特性・持ち物・技4つを登録。JSON形式でエクスポート/インポート可能。

## 3.7 ポケモンデータベース（FR-007）

3層アーキテクチャで管理（詳細は [データ設計](./06-data-design.md) を参照）。

| レイヤー                 | 内容                                       | 更新頻度       |
| ------------------------ | ------------------------------------------ | -------------- |
| Layer 1: ベース          | PokeAPIから取得した種族値・技・特性等      | 初回のみ       |
| Layer 2: Championsパッチ | 本家との差分（種族値変更、新メガシンカ等） | アップデート時 |
| Layer 3: シーズン定義    | 使用可能ポケモンリスト・ルール             | シーズン切替時 |
